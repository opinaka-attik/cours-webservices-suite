
## 2️⃣ TP — WEBHOOK

### Contexte

Un client génère des événements (nombres aléatoires).
Des serveurs webhook se sont abonnés pour recevoir uniquement certains types de valeurs (pair / impair).

### Énoncé

> 1. Lancez le client webhook et les serveurs abonnés.
> 2. Observez quels serveurs reçoivent quels événements.
> 3. Ajoutez une nouvelle règle de filtrage dans le client.
> 4. Ajoutez un nouveau serveur webhook correspondant à cette règle.
> 5. Relancez sans modifier les autres serveurs.

### Travail attendu

* Identifier **où se fait le filtrage**
* Expliquer comment un serveur “s’inscrit” pour recevoir des événements
* Décrire le rôle du client webhook

### Objectif 

Découvrir une **API Webhook (push événementiel)**.
