## 3️⃣ TP — SSE (Server-Sent Events)

### Contexte

Un serveur SSE envoie en continu des données aux clients connectés via une connexion HTTP persistante.

### Énoncé

> 1. Lancez le serveur SSE.
> 2. Connectez un premier client et observez les messages reçus.
> 3. Connectez un second client.
> 4. Rafraîchissez l’un des clients.
> 5. Observez les différences de comportement.

### Travail attendu

* Identifier **qui envoie les données**
* Déterminer si le client doit refaire des requêtes
* Expliquer ce qu’il se passe lors d’une reconnexion

### Objectif 

Découvrir une **API SSE (flux unidirectionnel en temps réel)**.
