
## 4️⃣ TP — WEBSOCKET

### Contexte

Un serveur WebSocket échange des messages en temps réel avec un client.
Le client transforme les données reçues (inversion de chaîne) et renvoie le résultat au serveur.

### Énoncé

> 1. Lancez le serveur et le client WebSocket.
> 2. Observez les échanges de messages dans les deux sens.
> 3. Modifiez le traitement effectué par le client.
> 4. Observez l’impact côté serveur.
> 5. Arrêtez puis redémarrez le serveur pendant l’exécution.

### Travail attendu

* Identifier le **sens des échanges**
* Déterminer si la communication est persistante
* Comparer ce comportement avec SSE

### Objectif 

Découvrir une **API WebSocket (communication bidirectionnelle temps réel)**.
